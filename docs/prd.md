# PRD — 의성김씨 아천문중 세보

## 1. 문서 목적
본 문서는 ‘의성김씨 아천문중 세보’ 프로젝트의 제품 요구사항을 정리한 문서이다.  
이 문서는 무엇을 만들지, 왜 필요한지, 누가 쓰는지, 어떤 기능과 화면이 핵심인지, 어떤 순서로 개발할지를 정의한다.

---

## 2. 제품 개요

### 제품명
- 의성김씨 아천문중 세보

### 부제
- 디지털 족보 및 문중 정보 시스템

### 한 줄 설명
구글 시트에 입력된 족보 자료를 기반으로, 문중원이 모바일 환경에서 자신의 계보와 가문의 흐름을 쉽게 탐색할 수 있도록 설계한 디지털 세보 앱.

---

## 3. 문제 정의
전통 종이 족보는 기록의 깊이는 크지만, 탐색성과 접근성이 낮다.

주요 문제:
- 원하는 인물을 빠르게 찾기 어렵다.
- 동명이인 구분이 어렵다.
- 직계 조상, 형제 관계, 분기 흐름을 한눈에 보기 어렵다.
- 세보와 역사 자료, 문헌 자료가 분리되어 있다.
- 모바일에서 보기 불편하다.
- 문중원 참여 기능이 부족하다.

---

## 4. 제품 목표
이 제품은 다음을 목표로 한다.

1. 본인 기준으로 계보를 쉽게 탐색할 수 있게 한다.
2. 직계 조상, 부계 형제, 세보 흐름을 시각적으로 보여준다.
3. 전통 족보 형식을 해치지 않으면서 디지털 탐색성을 높인다.
4. 역사 자료, 대동보 자료, 족보 이야기, 문중 참여 기능을 하나의 시스템에 통합한다.
5. 향후 1000~1500명 규모의 문중 데이터로 확장 가능한 구조를 만든다.

---

## 5. 대상 사용자
- 아천문중 문중원
- 족보 및 계보를 확인하려는 가족 구성원
- 문중 운영진 및 관리자
- 대동보, 선조 관련 자료를 열람하려는 사용자

---

## 6. 핵심 사용자 시나리오

### 시나리오 A: 본인 확인
사용자는 이름을 입력하고, 동명이인 중 아버지 이름을 보고 본인을 선택한다.  
앱은 선택된 인물을 기준으로 직계 요약, 세보 트리, 8촌 탐색의 기준점을 설정한다.

### 시나리오 B: 다른 문중원 조회
사용자는 본인을 기준으로 앱을 사용하되, 다른 문중원의 이름을 검색하여 그 사람의 카드, 조상표, 세보 위치, 관련 자료도 열람할 수 있다.

### 시나리오 C: 세보 트리 탐색
사용자는 1~24세 선조 요약, 25~30세 핵심 세보 트리, 31세 선택판, 32세 이후 하위 트리를 순차적으로 탐색한다.

### 시나리오 D: 촌수와 8촌 확인
사용자는 두 사람의 촌수 관계를 가로형 관계도로 확인하거나, 고조부 기준의 8촌 트리를 탐색한다.

### 시나리오 E: 문중 자료 열람
사용자는 아천문중 페이지에서 연표, 선조, 대동보 PDF, 족보 이야기, 정관·문중재산 자료, 문중원 투표 기능을 이용한다.

## 7. 주요 기능 요구사항

### 7.1 홈
- 상단 헤더(제목, 부제)
- 본인 확인 Bottom Sheet
- 문중 알림판
- 직계 요약 미리보기

### 7.2 본인 확인
- 이름 검색
- 동명이인 후보 표시
- 아버지 이름 기준 구분
- 선택한 인물 ID 저장
- 앱 상태 또는 브라우저 저장으로 재사용 가능

### 7.3 본인 카드
- 문중원 ID
- 이름
- 세손
- 배우자 이름
- 후손 표시
- 부/모, 조부/조모, 증조부/증조모, 고조부/고조모
- 여성 문중원은 외손 기준 표시
- 외손은 최대 2명 + `외 n명`

### 7.4 직계 조상 연결표
- 최대 30대까지 추적
- 직계 조상을 중심에 표시
- 배우자 대신 부계 형제 표시
- 형제 최대 8명
- 5~8대 기본, 30대까지 확장

### 7.5 세보 트리
- 1~24세: 족보식 카드 요약
- 25~30세: 4개 파 중심 핵심 트리
- 31세: 선택판
- 32세 이후: 선택형 하위 트리
- 직계 조상 라인은 굵은 선
- 확대/축소/초기화/중심 이동

### 7.6 촌수 계산
- 사람 2명 선택
- 공통 조상 계산
- 촌수 계산
- 공통 조상을 가운데 둔 가로형 관계도 표시

### 7.7 8촌 찾기
- 고조부까지 추적
- 고조부 기준 후손 집합 구성
- 8촌 범위를 트리로 표시
- 확대/축소/드래그 가능

### 7.8 선조의 발자취
- Leaflet 지도
- 세대 슬라이더
- 연도 병기 가능
- 장소 마커
- 마커 클릭 시 설명 팝업
- 이동 경로선 표시 가능

### 7.9 아천문중
보조 탭:
- 가문의 연표와 선조
- 대동보와 족보이야기
- 정관 / 문중재산
- 문중원 투표

### 7.10 문중원 투표
- 이름 입력
- 안건 선택
- 동일 안건에서 같은 이름은 1회만 허용
- 중복 시 차단 메시지 표시
- 간단 운영 우선, 강한 인증은 고려하지 않음

세부 동작 흐름(권장):
- 투표 화면 진입 시 `action=vote`로 안건/선택지/득표수를 로딩한다.
- 제출 전 `action=voteCheck&agendaId=&voterName=`로 중복 여부를 확인한다.
- 중복이 아니면 `action=voteSubmit`으로 저장하고 성공/실패 메시지를 표시한다.
- 저장은 `VoteResponse` 시트에 기록하며, 중복 판정은 `agendaId + voterName` 조합으로 한다.

## 9. 기술 요구사항

## 10. 비기능 요구사항

### 성능
- 초기 데이터 로딩 후 메모리 인덱싱
- 대규모 데이터에도 확장 가능한 세보 구조
- 31세 선택판을 활용해 후대 트리 복잡도 완화

### 운영성

## 12. 성공 기준
- 사용자가 본인을 검색하고 정확히 선택할 수 있다.
- 본인 카드와 직계 요약이 정상적으로 표시된다.
- 세보 트리의 1~24세, 25~30세, 31세, 32세 이후 구조가 분리되어 작동한다.
- 두 사람의 촌수 관계가 시각화된다.
- 선조의 발자취에서 세대별 장소를 확인할 수 있다.
- 아천문중 탭에서 역사·문헌·참여 기능을 이용할 수 있다.
