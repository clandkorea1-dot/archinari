# 신실행명세서 — 의성김씨 아천문중 세보

> 본 문서는 **각 페이지의 디자인과 기능을 수정/정의하는 “최우선 실행 기준”**이다.  
> 본 프로젝트의 모든 구현은 본 문서를 우선 참조한다.

## 1. 홈페이지

### 1) 헤더
- **헤더(맨 상단에 항상 표시되는 ‘디지털족보및~’)** 는 모든 페이지에 고정한다.
- 현재와 동일하다.

### 2) 홈 화면 박스 구성(총 4개)
현재 홈화면에는 3개의 큰 박스가 있는데 맨 아래에 하나 더 추가하여 총 4개의 박스를 만든다.
- 박스 안의 색상은 현재보다 **약간만 더 진하게** 하여 종이 족보의 느낌을 살린다.
- 박스들의 **테두리는 검은색 실선**으로 하여 분류가 쉽게 보이게 한다.

### 3) 첫 번째 큰 박스(현재 “세보검색”) 수정
현재 박스 전체의 틀은 유지하되, `UPDATE` 부분의 날짜는 **GOOGLE SHEET에 정보가 올라간 날짜**를 반영한다.

#### (1) 본인 확인 영역(성 제외 이름 → ID 찾기/확정)
- 성을 제외한 이름을 입력하면 문중원의 ID 후보를 찾는다.
- 동명이인이 있는 경우, **아버지 이름을 최대 8명까지 표시**(글자 크기 조정)하여 사용자가 구분 선택한다.
- 선택된 문중원의 데이터를 **캐시하여 저장**하고, 앱의 모든 데이터 정렬/기준을 이에 맞춘다.
- 검색 방식: `searchIndex`를 사용하여 이름으로 후보를 찾는다.
- 본인 확정: 후보를 선택하면 해당 인물의 `id`를 `selectedPersonId`로 저장한다.
- 본인 확인은 본인 기준 탐색(직계 요약, 세보 트리, 8촌 찾기, 촌수 계산)의 기준점을 설정하기 위한 기능이다.
- 확정된 ID는 앱 상태에 저장하며, 필요 시 브라우저 저장 기능을 이용해 재방문 시 유지한다.
- 단, 본인 기준 탐색 기능일 뿐이며 앱 전체에서는 다른 문중원의 자료도 함께 조회할 수 있다.

##### 기존 우측 3선 “선택바” 처리
- 현재 이 칸의 우측 3선의 선택바가 있는 칸은 **세보검색 칸과 합하고 메뉴바는 지운다.**
- 더 커진 이 칸의 용도는:
  1) (성 제외)이름으로 ID찾기  
  2) 동명이인 구분을 위한 아버지의 이름 표시/선택  
  3) 선택된 문중원의 기본 표시(아래 항목)

##### 선택된 문중원 “기본 표시” 항목
- 본인의 ID, 세손
- 아버지 이름, 어머니 이름
- 자녀 이름, 배우자 이름
- 여성 문중원: “외손” 열에 표시된 이름이 자녀 이름이다.
- 외손은 **최대 2명만 표시**(초과 시 `외 n명`)

#### (2) ‘검색’칸 용도 변경: “직계조상 찾기”
- 이 칸의 용도와 이름은 **직계조상 찾기**다(돋보기 단추).
- 돋보기를 누르면 현재 SEBO 박스 전체에 조회한 이름의 선조가 **시조까지**, **좌→우**로 차례로 표시된다.
- 아버지들의 이름은 **점선으로 연결**된다.
- 이름 아래에는 **세손 숫자를 괄호**로 넣어 표시한다.
- 따라서 현재 SEBO 박스의 이름은 **직계조상 찾기**로 바뀐다.

#### (3) 가계도 칸
- 이 칸을 누르면 **세계도 트리를 그리는 가계도 페이지**로 연결된다.

#### (4) 8촌 칸
- 아이콘을 누르면 홈페이지 맨 아래 새로 만든 박스에:
  - 조회된 문중원의 **8촌 명단(고조부 아래의 문중원만 표시)** 을
  - 세손수로 구분하여 표시한다.

#### (5) ARCHEON 칸
- 구글 `NOTICE` 시트와 연결된 **최신 3개 공지**의 제목 표시.
- 클릭 시 `<아천 문중>` 페이지 ‘공지사항’ 칸으로 연결.

### 4) 촌수계산 박스(용도 유지, 표시 개선)
현재 표시되는 “명세: 두 문중원~~~” 문구를 다음처럼 바꾼다.
- 문중원 1,2의 **공통조상**을 기준으로:
  - 공통 조상의 **이름과 세손을 파란색으로 구분**
  - 좌우 양쪽으로 **점선 연결**하여 시각적으로 표시
- 표시될 이름의 숫자는 최대 **36명(좌 18명, 우 18명)**.
- 문중원 1,2의 ID 입력 칸은 **하나로 두고 양쪽으로 구분**, 맨 끝에 “계산” 작용키를 둔다.

## 2. 가계도 페이지 명세
가계도 페이지는 D3.js 기반 세계도(세보) 트리를 사용한다.
- 1차 구현: **SVG**
- 2차 검토: **Canvas**(성능 필요 시)

가계도 페이지 안에는 보조 탭 또는 세그먼트 버튼으로 기능을 나눈다.

### 2-1. 세대 구간 선택
전체 세대를 한 화면에 동일 방식으로 표시하지 않고, 세대 구간별로 다른 표시 방식을 적용한다.

사용자가 선택할 구간:
- **선조 요약 구간(1세~24세)**
- **핵심 세보 트리 구간(25세~30세)**
- **31세 선택판**
- **후대 상세 트리 구간(32세 이후)**

표시 원칙:
- **1세~24세**: 전통 족보식 카드 및 분기 노드 중심 요약
- **25세~30세**: 4개 파(광택, 광진, 광룡, 광혁) 포함 핵심 트리
- **31세**: 선택판/카드 표
- **32세 이후**: 31세에서 선택된 인물을 root로 하는 하위 트리

본인 확인 완료 시:
- 선택 인물을 기준으로 직계 조상 연결선(부계 직계 라인)을 일반 선보다 **굵게 표시**한다.

### 2-2) 31세 선택판
- 31세는 별도 목록/카드/선택판으로 표시한다.
- 31세 인물 중 1명을 선택하면, 그 인물을 root로 하여 32세 이후 하위 트리를 다시 그린다.
- 선택 인물 기준 직계 조상 라인은 굵은 선으로 강조한다.

### 2-3) 트리 기능
트리 화면 기능:
- 확대/축소/초기화
- 접기/펼치기
- 특정 인물 선택 시 강조
- root 변경 후 re-render
- 중심 이동

### 2-4) 성별 시각 구분
성별에 따라 노드 테두리/색조를 다르게 표시한다(차분한 톤).
- 남성: 푸른 계열
- 여성: 분홍/보라 계열

### 2-5) 세보 트리 화면 구성(구간 상세)
한 화면에 전 세대를 펼치지 않고, 족보식 상단 요약 + 분기 트리 + 선택형 하위 트리로 구성한다.

#### 상단 선조 요약 구간(1세~24세)
- **1세~10세**: 시조~중시조 가로 카드 연결
- **11세~20세**: 11세의 3개 분기 노드를 포함한 카드 연결
- **21세~24세**: 세거지 분기와 후속 연결 요약

큰 카드 묶음(선택):
- 1세~10세
- 11세~20세
- 21세~24세

#### 11세 분기 표시
11세에서 3개 분기:
- 파조(평장사공파)
- 상서공파
- 기타 dd공파

내 조상 계통은 계속 연결하고, 나머지 2개 분기는 분기 노드로 표시하되 이후 기록은 없는 상태로 둔다.

#### 21세 세거지 분기
21세에서 4개 세거지 분기:
- 내 조상 계통 외 3개 분기도 이후 자료가 없어도 분기 노드로 표시한다.

#### 25세 이후 핵심 파 분기
25세에서 중요 4개 파 분기:
- 광택
- 광진
- 광룡
- 광혁

향후 확장을 고려해 4개 분기에 동일한 공간을 확보한다.

#### 25세~30세 트리
- 25세를 4개 파의 큰 분기 루트로 표시
- 26~30세는 각 파별 하위 가지 전개
- 광혁파는 실제 노드를 적극 반영
- 나머지 3개 파는 최소 반영 + 확장 공간 확보

#### 31세 선택판
- 31세는 17명을 표/카드로 표시한다.
- 선택 시 해당 인물 root로 32세 이후 하위 트리를 다시 그린다.

#### 32세 이후 하위 트리
인원 급증을 고려하여 31세 선택 후 하위 트리를 표시한다.
- 확대/축소/중심 이동/초기화/재정렬

## 3. 선조의 발자취 페이지 명세
Leaflet.js 기반. 역사상 선조의 이동 경로/거주 흔적을 시각화한다.

기능:
- 위도/경도 기반 마커 표시
- 세대 슬라이더 제공
- 세대 슬라이더에 가능한 경우 **연도 병기 준비**
- 세손의 장소 마커
- 마커 클릭 시 설명이 지도 아래 별도 박스에 표시(간략 설명 박스)
- 순서 정보가 있으면 이동 경로선 표시 가능

별도 박스 #1 표시 항목:
- 인물명
- 세손 또는 연도
- 간단 설명

별도 박스 #2 추가(구분):
- **가문의 연표와 선조**
- **대동보와 족보이야기**

데이터 구조 원칙:
- 선조의 발자취 데이터는 사람 기본 데이터와 분리된 `Movement` 시트를 권장한다.

## 4. 아천문중 페이지
문중 관련 자료를 함께 볼 수 있는 페이지.

보조 탭 구성:
1) **정관 / 문중재산**
2) 공지사항: `notice` 시트와 연결
3) **문중원 투표**

### 문중원 투표
입력/선택:
- 이름
- 투표 항목 선택

중복 방지:
- 동일 안건에서 같은 이름이 이미 시트에 기록되어 있으면 재투표를 막는다.
- 이미 투표한 경우 “이미 참여하셨습니다” 안내 표시.

저장 시트:
- `VoteResponse`

비고:
- 동명이인 구분/강한 인증은 고려하지 않는다. 보안보다 편의/간단 운영 우선(1차 방식).

## 5. PWA/캐시 전략과 실행명세서

### 5-1) 프로젝트 개요
Google Sheets → Apps Script(JSON API) → PWA 프론트엔드 → GitHub/Netlify 배포

역할:
- 구글 시트: 원본 데이터 저장소
- Apps Script: 데이터 가공/JSON API
- PWA 프론트: 사용자 화면
- GitHub: 버전 관리
- Netlify: 정적 프런트 배포

앱 제목/부제:
- **제목**: 의성김씨 아천문중 세보
- **부제**: 디지털 족보 및 문중 정보 시스템

### 5-2) 개발 원칙
#### 데이터와 화면 분리
데이터는 구글 시트/Apps Script, 화면은 정적 프런트(Netlify)로 분리한다.

#### 모바일 우선
데스크톱보다 모바일 사용성을 우선한다.

#### PWA 적용
- `manifest.json`
- `service worker`
- app shell 구조
- 정적 자원 캐시

데이터는 최신 반영을 위해 **network-first**.

#### 성능 최적화(메모리 인덱싱)
초기 로딩 후 메모리 인덱스 생성.

필수 인덱스:
- `peopleById`
- `childrenByFather`
- `searchIndex`
- `peopleByGeneration`

추가(필요 시):
- `siblingsByFather`
- `peopleByName`
- `spousesByPerson`

캐시 전략:
- 정적 파일(HTML/CSS/JS/아이콘): **cache-first**
- 데이터 API(people/notices/movements/history/voteResponse): **network-first**

### 5-3) 배포/운영
- GitHub: 버전 관리/이력/롤백
- Netlify: GitHub 연동 자동 배포
- Apps Script: 구글 시트 연결 JSON API 제공

### 5-4) main.js 최적화(고속 엔진 연계)
속도 저하 개선책:
1. 검색 결과 리스트는 서버가 이미 포함해 보내는 `fatherName`을 **그대로 사용**해 재조회 제거
2. `getCache()` 로직을 프런트에도 적용: 한 번 불러온 인물 정보는 메모리에 저장하여 재조회 시 즉시 표시
3. `fetch` 호출 최소화: 가계도(D3) 그릴 때 필요한 데이터만 요청하도록 로직 최적화
4. 결과 카드 표시 시 **부드러운 애니메이션**으로 체감 속도 개선
