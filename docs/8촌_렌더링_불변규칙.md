# 8촌 친척 시각화 — 불변 규칙 (고정 사양)

이 문서는 **홈「8촌 친척」영역** 및 **8촌 관계도(`eight-kin-svg`)**에서 쓰는 **표현 방식(레이아웃·기하·정렬)**을 보존하기 위한 기준입니다.  
**구조·동작을 바꾸는 수정은 하지 않습니다.** 색상만 디자인 토큰과 맞출 수 있으며, 연결선 색은 **seal 녹색(`#166534`) 계열**로 통일합니다 (`main.js`의 `EIGHT_KIN_EDGE`, `EIGHT_KIN_EDGE_SOFT`).

---

## 1. 가로 가계도 (`mountEightKinHorizontalTreeSvg`)

- **열(col)**: 왼쪽이 기준(앵커)·오른쪽으로 세대(세손 기준 열). 열 너비 `COL_W`, 좌우 패딩 고정.
- **세대 내 정렬**
  - 한 열 안에서 **부친 ID 오름차순**으로 그룹을 배치.
  - 같은 부친 아래 자녀는 **문중원 ID 오름차순**.
  - 기준 인물(앵커)은 해당 열에서 **맨 앞**으로.
- **가족 그룹 배치**
  - `fatherId`가 같으면 같은 그룹. 없으면 1인 그룹(`S:자신id`).
  - **같은 아버지 자녀는 반드시 연속 블록**으로 세로 배치(형제가 떨어지지 않음).
  - 그룹 순서는 **이전 열(아버지 세대)의 세로 순서**를 우선 반영하고, 없으면 부친 ID 문자열 순.
- **아버지 스텁**: 목록에 부친 노드가 없어도 연결을 위해 **임시 부 노드**를 만든다(`ensureFatherStubs`).
- **이름 텍스트**
  - `text-anchor: start`, x는 `n.x - n.w/2` (열 중심 기준 **좌측 정렬**).
  - 일반 이름 색: 본문 톤 `#1c1917`, 기준 인물: `#166534`.
- **연결 기하 (형제·부자)**
  - 같은 부친의 자녀가 둘 이상: 자녀 이름 **왼쪽 끝 기준**으로 한 세로선 상의 **`xDot = min(이름왼쪽) - 12`** 에 **점(•)** 을 찍는다.
  - **점 ↔ 이름 사이에는 선을 그리지 않는다** (가로 “긴 줄” 방지).
  - 부친 오른쪽(`p.x + p.w/2 + 6`)에서 **`xDot`, 세로 중앙 `yMid`** 로 **베지어 곡선** 한 줄.
  - 형제 점들의 **y는 각 자녀의 이름 baseline 높이**와 동일.
  - 형제가 둘 이상이면 점들을 **`xDot` 고정에서 `minY`–`maxY` 수직선**으로 잇는다 (형제 “버스” 축).
- **열 전체 세로 정렬**: 인원이 가장 많은 열의 세로 중앙을 기준으로 다른 열을 **센터 이동**한 뒤, 위로 밀리면 전체를 아래로 보정.

## 2. 층형 안내용 SVG (`mountEightKinTreeSvg`)

- 세대 **층** 사이를 잇는 짧은 연결선은 **8촌 톤**(seal 계열 연한색)으로 통일한다. (레이아웃·층 구조 자체는 변경 금지.)

---

## 3. D3 관계도 (`renderEightKinTree`)

- **트리형 데이터**일 때: `paintD3TreeLayout`을 사용한다. **SVG `id === "eight-kin-svg"`** 인 경우에만 링크 색을 8촌 seal 계열로 한다 (가계도 탭 `tree-svg`는 기존 회색 링크 유지).
- **포스 그래프**일 때: 링크(선) 색을 8촌 seal 계열로 통일한다.

---

## 4. 색상 (가변 허용 범위)

- **연결선·점·형제 수직선**: `#166534` 및 `rgba(22, 101, 52, …)` 계열만 사용. 청록·하늘색 계열 사용 금지.
- **이름·카피 문구**: 기존 앱 타이포·seal 규칙과 동일.

---
