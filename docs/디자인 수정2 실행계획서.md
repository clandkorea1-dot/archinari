## 개요
- **목표**: Google Sheets(Apps Script API) 데이터를 기반으로, 현재 PWA(바닐라 JS + D3) UI/UX를 `디자인 수정2.pdf` 요구사항에 맞게 반영한다.
- **원칙**: React/React-flow로 전환하지 않고, **현재 구조 유지 + D3/DOM만 개선**한다.
- **성능 기준(체감)**: 8촌/촌수 기능은 **화면 렌더를 막지 않도록** 단계적 렌더(먼저 표시 → 이후 보강)와 버튼 기반 작업(필요 시만 시각화)을 적용한다.

## 현재 구현 상태 요약(2026-04-15)
- **홈**: 본인확인, 직계 조상 연결, **8촌 친척**, **촌수 계산** 섹션 순(위→아래).
- **헤더**: 검은 실선 테두리/구분선 + 페이지별 서브메뉴(가계도/발자취/아천문중) 표시 및 동작 일부 연결.
- **8촌 친척**: 왼쪽 기준(고조부) → 오른쪽 세대 열, **부친ID가 있을 때** seal 녹색 계열 실선으로 부-자·형제 연결. 형제(동일 부친) 묶음 버스(곡선/세로줄) 적용. **기하·정렬 불변 사양**: `docs/8촌_렌더링_불변규칙.md`.
- **성능**: 8촌은 고조부 추정용 직계 체인만 짧게 조회 후 렌더, 긴 직계(시조까지)는 백그라운드 보강. 촌수 관계도는 버튼 클릭 시에만 렌더.

## 요구사항 체크리스트(디자인 수정2.pdf)
### 1. 홈 페이지
1) **박스 바탕색**: 홈 각 박스의 바탕색을 헤더 바탕색과 동일하게.
- 상태: 적용(홈 `.paper-box` 배경을 헤더와 동일 톤으로 조정).

2) **선 교체**:  
- 헤더 테두리/칸 구분선: 하단 박스에서 쓰던 **검은 실선**처럼 선명하게  
- 하단 박스 테두리: 헤더의 연한 테두리 톤으로
- 상태: 적용(헤더는 `hdr-frame/hdr-divider/hdr-vdiv` 검은 실선, 박스 테두리는 연하게).

3) **동명이인 카드 디자인 변경**:
- 예시: `선영 34세손 · 문중원ID 235` / `부친: 동구  탭하여 상세 보기`
- 상태: 적용(검색 결과 카드 레이아웃 변경).

4) **선택된 기준 인물 패널**
- (1) `형제자매` 라벨을 형제 이름 위로 위치 조정
- (2) 자녀 최대 8명, 초과 시 `외 N명`
- (3) 작은 `가계도/8촌/공지` 박스 제거
- 상태: 적용(라벨 위치/자녀 제한/3버튼 제거).

5) **촌수 계산 박스**
- 상단 설명 문구 변경: `ID로 검색하면 공통 조상 기준 좌우로 연결 시각화`
- 결과 문구 단순화: `촌수: 숫자 (공통 조상 이름, 세손 숫자)` 형태
- 상태: 적용(상단 문구/결과 포맷/관계도는 버튼 클릭 시만).

6) **8촌 친척 박스**
- 중복 안내 문구 제거
- 고조부의 **직계 후손만** 표시(고조부 형제 자손 제외)
- 같은 아버지의 자녀는 곡선/묶음으로 보기 좋게 연결
- 상태: 부분 적용
  - **연결선/묶음**: 적용(seal 녹색 실선 + 버스). 불변 규칙은 `docs/8촌_렌더링_불변규칙.md`.
  - **직계 후손만 필터**: `eightKin` 응답의 `부친ID` 체인이 충분히 일관적일 때만 안정적으로 가능. 현재는 데이터 스키마/값 불일치 가능성이 있어 “가능한 연결” 중심으로 동작하도록 보호 로직이 들어가 있음.

### 2. 가계도/발자취/아천문중 페이지
- 헤더 디자인은 동일하되, **페이지별 서브메뉴**를 넣어 구분
  - 가계도: `9-11세 / 11-20세 / 21-31세 / 32세 이후`
  - 발자취: `가문의 연표와 선조 / 대동보와 족보이야기`
  - 아천문중: `정관 / 문중재산 / 공지사항 / 문중원투표`
- 상태: 적용(표시 + 기본 동작 연결)
  - 가계도 서브메뉴: 세대 필터로 인물 목록 필터링/갱신
  - 발자취/아천문중 서브메뉴: 섹션으로 스크롤 이동

## 남은 작업(우선순위)
### A. 8촌 친척 “직계 후손만”을 100% 정확하게
- **요구**: 고조부(1명) 기준, 고조부 형제 방계는 제외.
- **조건**: `eightKin` 항목에 아래 필드가 안정적으로 존재해야 함
  - **본인 문중원ID**
  - **부친 문중원ID(부친ID/fatherId/fatId/아버지의ID 등)**
  - **세대(세손)**
- **계획**:
  - Apps Script `eightKin` 응답에 `id`, `fatherId`, `gen`을 **필수로 포함**하도록 스키마 고정
  - 프론트는 `id -> fatherId` 체인으로 `anchorId(고조부)`에 도달하는 항목만 남김

### B. 촌수 계산 성능 최적화(서버)
- **문제**: 서버가 `running`만 반환하거나 계산 시간이 길면 UX가 저하됨.
- **계획**:
  - Apps Script `kinship`가 **최종 `distance/촌수`**를 반드시 반환하도록 개선
  - (가능하면) 서버에서 공통조상 및 세대 정보를 함께 반환하여, 프론트 추가 조회 최소화

### C. 헤더 서브메뉴의 “실제 콘텐츠” 연결 강화
- 발자취/아천문중은 아직 “스크롤 이동” 중심이므로, 실제 섹션 구성 확정 시 ID/앵커를 정교화.

## 테스트 체크리스트
- **홈 검색**: 동명이인 8명까지 카드가 예시 디자인으로 표시되는지
- **기준 인물 선택 패널**: 형제 라벨/자녀 8명 제한/버튼 제거 확인
- **직계 조상 연결**: 본인→시조(또는 가능한 최상단) 표시가 끊기지 않는지
- **촌수 계산**: 결과 텍스트만 먼저 나오고, 관계도는 버튼 클릭 시에만 그려지는지
- **8촌 친척**: 확대/축소/드래그 동작, 부친ID가 있는 경우 선이 보이는지, 형제 묶음이 적용되는지
- **헤더 서브메뉴**: 탭 전환 시 서브메뉴가 맞게 바뀌고 버튼이 동작하는지
